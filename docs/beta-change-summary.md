# β版変更サマリー

このアプリは株価・業績・ニュース・メモを整理する調査補助ツールです。投資判断や売買推奨を目的にしていません。

## β版までに実装した主要機能

- 銘柄登録、編集、削除
- 株価CSVテンプレート、プレビュー、取り込み
- 株価チャート、移動平均線、トレンドスコア
- 業績CSVテンプレート、プレビュー、取り込み
- 成長率、成長性スコア、財務安全性スコア、総合調査スコア
- ウォッチリスト、確認水準メモ
- リスク管理、リスクスコア
- ニュース・材料メモ管理
- 決算カレンダー
- 確認タスク管理
- Mock APIによる株価、業績、ニュース、決算予定更新
- 実LLM API Route
- Mock LLM
- AI分析生成、保存、履歴表示
- AI分析履歴の検索、比較、差分表示
- Markdown/JSONエクスポート
- LLM送信設定保存
- LLM利用ログ、回数制限、入力サイズ制御
- CSVインポート履歴
- 株価データだけ削除、業績データだけ削除

## UI情報設計改善

β版前に、情報量の多い画面を中心に見直しました。

- ダッシュボードで次に見るべき項目を上部へ整理
- 銘柄一覧のスコア、ニュース、タスク状況を見やすく整理
- CSV取り込みをテンプレート、入力、プレビュー、実行の流れへ整理
- 銘柄詳細のニュース、リスク、決算、タスクを要約優先に変更
- 履歴、ログ、LLM入力プレビューを下部に集約
- Danger Zoneを最下部に分離
- AI履歴比較では、変化あり項目を優先表示

実LLM APIはE2Eで呼びません。

## localStorageバックアップ/復元

設定画面に「β版データバックアップ」を追加しました。

- `stock-trend-mvp:` で始まるキーだけをJSON化
- バックアップ日時とschemaVersionを保存
- APIキーらしい値を検出した場合は中止
- 復元前に対象キーを表示
- 確認チェック後に上書き復元
- 対象外キーは復元しない

## 未確認の点

- 実LLM APIの本番利用時の費用・レート制限
- 大量データ投入時のlocalStorage容量上限
- 複数端末での同期
- 長期間運用した場合のAI履歴・CSV履歴の肥大化

## 今後の課題

1. Git管理または定期ZIPバックアップの運用を固定する
2. localStorage容量が増えた場合の履歴整理機能を追加する
3. 実株価API・実業績APIをサーバー側で安全に接続する
4. バックアップJSONの差分比較や世代管理を追加する
5. サーバーDB保存や複数端末同期を検討する
