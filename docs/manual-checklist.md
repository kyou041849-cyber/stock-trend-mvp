# β版 手動確認チェックリスト

このチェックリストは、ブラウザで株価・ファンダメンタル分析アプリを確認するためのものです。

このアプリは調査補助ツールです。スコアやAI分析は機械的な整理であり、投資判断ではありません。

## 起動

1. `npm run dev` を実行する。
2. `http://127.0.0.1:3000` を開く。
3. 3000番が別アプリで使われている場合は `npm run dev -- -p 3001` を実行し、`http://127.0.0.1:3001` を開く。
4. 初回起動でサンプルデータが表示されることを確認する。
5. リロード後もデータが残ることを確認する。

## Playwright smoke test

手動確認の前に、主要導線だけを自動で確認できます。

```powershell
npm run test:e2e
```

このテストは `http://127.0.0.1:3010` を一時的に使い、Edgeで次を確認します。

- アプリ起動
- 銘柄登録
- 登録銘柄の詳細表示
- 株価CSVプレビュー
- 業績CSVプレビュー
- AI分析セクション表示
- AI履歴画面表示
- 設定画面表示

実LLM APIは呼びません。ポートが衝突する場合は以下のように変更できます。

```powershell
$env:E2E_PORT="3011"
npm run test:e2e
```

Playwright smoke testは「壊れていないか」の確認用です。見た目の読みやすさ、スマホ幅、文言の自然さは、この手動チェックリストで確認してください。

## localStorage耐性

1. localStorageを空にしてリロードし、クラッシュしないことを確認する。
2. 古い形式のデータがあっても画面が落ちないことを確認する。
3. 不正JSONを入れた場合でも、空状態またはデータ不足表示になることを確認する。
4. AI分析履歴、CSV履歴、LLM送信設定、LLM利用ログが空でも表示が壊れないことを確認する。

## 銘柄管理

1. 銘柄を登録できることを確認する。
2. 銘柄を編集できることを確認する。
3. 銘柄を削除できることを確認する。
4. リロード後も登録内容が保持されることを確認する。

## 株価データ

1. 株価CSVテンプレートをダウンロードできることを確認する。
2. 株価CSVを貼り付け、プレビューが表示されることを確認する。
3. 読み込み件数、追加予定、更新予定、エラー件数、警告件数が表示されることを確認する。
4. 不正な日付や数値で行番号付きエラーが出ることを確認する。
5. 「取り込み実行」を押すまで保存されないことを確認する。
6. 正常CSVを取り込むと、チャートと移動平均線が表示されることを確認する。
7. 株価Mock API更新が動くことを確認する。
8. 株価データだけ削除でき、銘柄情報や業績データが残ることを確認する。

## 業績データ

1. 業績CSVテンプレートをダウンロードできることを確認する。
2. 業績CSVを貼り付け、プレビューが表示されることを確認する。
3. 同じ会計年度が更新予定として表示されることを確認する。
4. 不正な数値で行番号付きエラーが出ることを確認する。
5. 正常CSVを取り込むと、業績表、グラフ、成長率、各スコアが更新されることを確認する。
6. 業績Mock API更新が動くことを確認する。
7. 業績データだけ削除でき、銘柄情報や株価データが残ることを確認する。

## ウォッチリスト・リスク・メモ

1. ウォッチリストへ追加、編集、削除できることを確認する。
2. 監視ステータス、調査優先度、次回確認日を保存できることを確認する。
3. 確認水準メモを保存できることを確認する。
4. リスクメモを追加、編集、削除できることを確認する。
5. リスクスコアが表示されることを確認する。
6. 調査メモ履歴を追加、編集、削除できることを確認する。

## ニュース・決算予定・タスク

1. ニュースを手入力できることを確認する。
2. ニュースMock取得でニュースが追加されることを確認する。
3. ニュースの重要度、材料分類、確認済みを編集できることを確認する。
4. 決算予定を手入力できることを確認する。
5. 決算予定Mock取得で予定が追加されることを確認する。
6. 決算予定から確認タスクを作成できることを確認する。
7. 確認タスクを作成、編集、削除できることを確認する。
8. ダッシュボードに近日中の決算予定やタスクが表示されることを確認する。

## AI分析

1. LLM入力プレビューにAPIキー、APIベースURL、認証情報、localStorageの生データが含まれないことを確認する。
2. Mock LLMでAI分析を生成できることを確認する。
3. AI分析を保存できることを確認する。
4. structuredReport形式のAI分析がセクション単位で表示されることを確認する。
5. contextHashの鮮度表示が出ることを確認する。
6. AI分析をコピーできることを確認する。
7. AI分析履歴の検索、絞り込み、並び替えが動くことを確認する。
8. 同じ銘柄のAI分析2件を比較できることを確認する。
9. 差分サマリー、追加項目、削除項目、変化なし項目が表示されることを確認する。
10. Markdown/JSONエクスポートが動くことを確認する。
11. 実LLMモードでAPIキー未設定の場合、安全にエラー表示されることを確認する。
12. 実LLMを使う場合は `.env.local` に `OPENAI_API_KEY` と `OPENAI_MODEL` を設定し、サーバー側API Route経由で動くことを確認する。
13. LLM送信項目のON/OFFで入力サイズが変わることを確認する。
14. 利用ログと回数制限が表示されることを確認する。

## CSVインポート履歴

1. 株価CSV取り込み後にCSVインポート履歴が表示されることを確認する。
2. 業績CSV取り込み後にCSVインポート履歴が表示されることを確認する。
3. 履歴に日時、銘柄、データ種別、ファイル名、追加件数、更新件数、エラー件数、成功/失敗が表示されることを確認する。
4. CSV本文全文が履歴に保存されていないことを確認する。

## 表示異常

1. 画面上に `NaN`、`Infinity`、`undefined`、`[object Object]` が表示されないことを確認する。
2. データ不足時は「データ不足」や `-` などの表示になることを確認する。
3. 壊れた日付や壊れた数値表示がないことを確認する。
4. スマホ幅で主要画面が大きく崩れないことを確認する。

## APIキー安全確認

1. `.env.local` が `.gitignore` 対象であることを確認する。
2. `.env.local.example` にサンプル値のみがあることを確認する。
3. APIキーが画面、localStorage、利用ログ、AI入力プレビューに保存・表示されないことを確認する。
4. 実LLMのAPIキーはサーバー側でのみ参照されることを確認する。

## 文言

1. 画面やAI出力が調査補助の表現になっていることを確認する。
2. 売買推奨や将来株価の断定に見える文言がないことを確認する。
3. AI分析に「この分析は入力データをもとにした調査補助であり、投資判断ではありません。」という注意文が含まれることを確認する。
