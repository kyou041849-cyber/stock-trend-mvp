# 初心者向け 使い方説明書

この説明書は、株価・ファンダメンタル分析アプリを初めて使う方向けのものです。

専門用語をできるだけ少なくして、まず何をすればよいかが分かるようにまとめています。

## まず知っておくこと

このアプリは、銘柄を調べるためのメモ帳兼チェックツールです。

株価、業績、ニュース、決算予定、リスクメモ、AI分析結果などを1つの場所で整理できます。

ただし、このアプリは投資判断をするものではありません。  
画面に表示されるスコアやAI分析は、入力されたデータを機械的に整理したものです。

## このアプリでできること

- 銘柄を登録する
- 株価CSVを読み込む
- 株価チャートと移動平均線を見る
- 業績CSVを読み込む
- 成長率やスコアを見る
- ウォッチリストで気になる銘柄を管理する
- リスクメモやニュースメモを残す
- 決算予定や確認タスクを管理する
- Mock AIまたは実LLMで調査メモを整理する
- AI分析履歴を検索、比較、エクスポートする

## 起動方法

PowerShellを開いて、アプリのフォルダへ移動します。

```powershell
cd "C:\Users\kyou0\Documents\日常の困りごと\stock-trend-mvp"
```

初回だけ、必要な部品を入れます。

```powershell
npm.cmd install
```

アプリを起動します。

```powershell
npm.cmd run dev
```

ブラウザで次のURLを開きます。

```text
http://127.0.0.1:3000
```

もし別のアプリが開いてしまう場合は、3000番を他のアプリが使っています。  
その場合は、別の番号で起動します。

```powershell
npm.cmd run dev -- -p 3001
```

そして、次を開きます。

```text
http://127.0.0.1:3001
```

## 画面の基本

最初の画面には、登録済みの銘柄一覧が表示されます。

主に使う画面は次の4つです。

- 銘柄一覧
- 銘柄詳細
- 設定
- AI分析履歴

まずは銘柄一覧から銘柄を選び、詳細画面で株価や業績を確認します。

## 1. 銘柄を登録する

1. 銘柄一覧画面で「新規登録」または追加ボタンを押します。
2. ティッカーを入力します。
3. 会社名、市場、セクター、メモを入力します。
4. 保存します。

入力例:

```text
ティッカー: TEST
会社名: テスト株式会社
市場: 東証
セクター: 情報通信
```

登録した銘柄は、あとから編集や削除ができます。

## 2. 株価CSVを読み込む

株価データはCSVで読み込めます。

まず、銘柄詳細画面または一覧画面から「株価CSV」を開きます。

### テンプレートを使う

「株価CSVテンプレート」を押すと、CSVの見本をダウンロードできます。

株価CSVの列は次の通りです。

```csv
date,open,high,low,close,volume
2026-01-05,100,105,98,103,1200000
2026-01-06,103,108,102,107,1500000
```

意味は次の通りです。

- `date`: 日付
- `open`: 始値
- `high`: 高値
- `low`: 安値
- `close`: 終値
- `volume`: 出来高

### 取り込みの流れ

1. CSVファイルを選ぶ、またはCSV欄に貼り付けます。
2. プレビューが表示されます。
3. 読み込み件数、追加予定、更新予定、エラー件数を確認します。
4. 問題なければ「取り込み実行」を押します。

同じ日付のデータがある場合は、後から読み込んだデータで更新されます。

エラーがある場合は保存されません。  
表示された行番号を見てCSVを修正してください。

## 3. 株価チャートを見る

株価CSVを取り込むと、銘柄詳細画面でチャートを見られます。

表示されるもの:

- 終値の折れ線チャート
- 出来高
- 25日移動平均線
- 75日移動平均線
- 200日移動平均線
- トレンドスコア

データが少ない場合、長い移動平均線は表示されないことがあります。  
たとえば200日移動平均線を見るには、200日分以上の株価データが必要です。

## 4. 業績CSVを読み込む

業績データもCSVで読み込めます。

銘柄詳細画面から「業績」を開きます。

### テンプレートを使う

「業績CSVテンプレート」を押すと、CSVの見本をダウンロードできます。

業績CSVの列は次の通りです。

```csv
fiscalYear,revenue,operatingIncome,netIncome,eps,operatingCashFlow,freeCashFlow,equityRatio,roe,roic,marketCap,per,pbr,psr
2022,100000,10000,7000,70,12000,8000,55,12,9,500000,30,3,5
2023,120000,14000,9000,90,15000,10000,58,14,10,650000,35,4,5.4
```

最初は、分かる項目から入力して構いません。  
ただし、次の項目は必須です。

- `fiscalYear`
- `revenue`
- `operatingIncome`
- `netIncome`

### 取り込みの流れ

1. CSVファイルを選ぶ、またはCSV欄に貼り付けます。
2. プレビューを確認します。
3. 追加予定、更新予定、エラー件数を確認します。
4. 問題なければ「取り込み実行」を押します。

同じ会計年度のデータがある場合は、後から読み込んだデータで更新されます。

## 5. スコアを見る

銘柄詳細画面では、いくつかのスコアが表示されます。

- トレンドスコア
- 成長性スコア
- 財務安全性スコア
- 総合調査スコア
- リスクスコア

これらは、入力データをもとにした機械的な集計です。  
スコアが高い、低いという表示だけで判断せず、元データやメモを確認してください。

## 6. ウォッチリストを使う

気になる銘柄はウォッチリストで管理できます。

登録できる内容:

- 監視ステータス
- 調査優先度
- 次回確認日
- 監視理由
- 次に確認すること
- メモ

「次回確認日」を入れておくと、あとで確認すべき銘柄を見つけやすくなります。

## 7. リスクメモを書く

銘柄ごとにリスクをメモできます。

例:

- 競合リスク
- 技術リスク
- 規制リスク
- 為替リスク
- 業績悪化リスク

リスクには、影響度と発生可能性を設定できます。  
それをもとに、リスクスコアが機械的に計算されます。

## 8. ニュース・決算予定・タスクを管理する

ニュース欄には、気になったニュースや材料を保存できます。

決算カレンダーでは、次の決算予定日を管理できます。

確認タスクでは、次に見るべきことをToDoとして残せます。

例:

```text
タスク名: 次回決算の売上成長率を確認する
期限: 2026-08-10
優先度: 高
```

## 9. AI分析を使う

AI分析では、銘柄の情報を整理したレポートを作れます。

まずはMock LLMで試すのがおすすめです。  
Mock LLMは実際の外部AI APIを呼ばないため、APIキーなしで動作確認できます。

できること:

- 銘柄要約
- ニュース要約
- 決算要約
- リスク整理
- 追加調査ポイント

AI分析には必ず注意文が付きます。

```text
この分析は入力データをもとにした調査補助であり、投資判断ではありません。
```

## 10. 実LLMを使う場合

実LLMを使う場合は、`.env.local` にAPIキーを設定します。

`.env.local.example` をコピーして、`.env.local` を作成してください。

```text
OPENAI_API_KEY=your-openai-api-key
OPENAI_MODEL=gpt-5-mini
```

APIキーはブラウザ側やlocalStorageには保存されません。  
サーバー側のAPI Routeからだけ使われます。

実LLMは利用回数制限があります。  
上限に達した場合は、Mock LLMを使うか、翌日に再度試してください。

## 11. AI分析履歴を見る

保存したAI分析は履歴から確認できます。

できること:

- 銘柄名やティッカーで検索
- 分析タイプで絞り込み
- キーワード検索
- 2件のAI分析を比較
- 差分表示
- Markdownでエクスポート
- JSONでエクスポート

同じ銘柄の分析を2件選ぶと、前回から変わった点を確認できます。

## 12. CSVインポート履歴を見る

銘柄詳細画面にCSVインポート履歴が表示されます。

確認できる内容:

- 日時
- 銘柄
- データ種別
- ファイル名
- 追加件数
- 更新件数
- エラー件数
- 成功または失敗

CSV本文全文は保存されません。

## 13. 株価データや業績データだけ削除する

銘柄詳細画面で、次の操作ができます。

- 株価データだけ削除
- 業績データだけ削除

銘柄情報、ニュース、リスクメモ、AI分析履歴などは削除されません。

CSVやMock APIでデータを入れ直したいときに使います。

## 14. データはどこに保存されるか

このアプリのデータは、使っているブラウザのlocalStorageに保存されます。

注意点:

- 別のブラウザでは同じデータは見えません。
- ブラウザのデータを削除すると、アプリのデータも消えることがあります。
- 重要なメモを長く残したい場合は、将来的にバックアップ機能を使うか、別の場所にも控えてください。

## 15. 困ったとき

### `npm` が使えない

PowerShellでは、次のように `npm.cmd` を使ってください。

```powershell
npm.cmd run dev
```

### `npm.ps1` の実行が無効と出る

PowerShellの実行ポリシーによる表示です。  
この場合も `npm.cmd` を使うと回避できます。

```powershell
npm.cmd install
npm.cmd run dev
```

### `127.0.0.1:3000` で別のアプリが開く

別のアプリが3000番を使っています。  
3001番など別のポートで起動してください。

```powershell
npm.cmd run dev -- -p 3001
```

### CSVでエラーが出る

プレビューに表示された行番号を確認してください。

よくある原因:

- 日付が `YYYY-MM-DD` になっていない
- 数値欄に文字が入っている
- 必須列が足りない
- 空欄にしてはいけない項目が空欄

### AI分析が実行できない

まずMock LLMで試してください。

実LLMを使う場合は、`.env.local` に `OPENAI_API_KEY` と `OPENAI_MODEL` があるか確認してください。

## 16. β版で特に確認してほしいこと

β版では、次の点を重点的に確認してください。

- CSVプレビューが分かりやすいか
- エラー行の説明が分かりやすいか
- AI分析履歴の検索や比較が使いやすいか
- リロード後もデータが残るか
- 画面に `NaN`、`Infinity`、`undefined` が出ないか
- 投資判断に見える表現が出ていないか

より細かい確認手順は、次のファイルにあります。

```text
docs/manual-checklist.md
```
